# 无尽深渊 - 项目状态（中期）

更新时间：2026-02-27

## 一、当前阶段结论
- 项目已完成可演示闭环：`主菜单 -> 开始 -> 游玩 -> 死亡结算 -> 返回主菜单/重开`。
- 核心玩法可运行，且关键中断问题（输入冲突、返回主菜单黑屏/蓝屏、页面错位）已完成修复。
- 中期评估完整度（功能向）：约 **80%-85%**。

## 二、已完成内容（截至 2026-02-27）

### 1) 核心玩法与稳定性
- 2D 纵向下坠跑酷主循环可运行：移动、碰撞、得分、掉血、失败、重开。
- 玩家为“仅左右移动 + 下坠触敌刹车反冲”模式，无主动攻击。
- 高速下落场景下的碰撞稳定性已做防穿透处理。
- 关卡段使用对象池回收，基础性能路径可用。

### 2) 主菜单与页面体系
- 主菜单已具备多页面入口与返回：
  - 排行榜（Leaderboard）
  - 收藏页（Collection）
  - 成就页（Achievements）
  - 设置页（Settings）
- 所有页面均已接入“返回主界面”按钮与“关闭”按钮。
- 页面切换逻辑统一，避免多面板同时显示。

### 3) 本地存储系统（已接入）
- 已建立统一本地进度存档：`run_progress.json`（位于 `Application.persistentDataPath`）。
- 已存储/读取内容包括：
  - 最佳分、最近分、总局数、总分
  - 排行榜记录（含模式与时间）
  - 模式解锁状态与当前选择模式
  - 收藏解锁状态与每个收藏物累计获取次数

### 4) 收藏系统（按固定深度刷新）
- 已将随机黄色颗粒式收集物替换为“固定深度里程碑收藏物”。
- 当前里程碑深度：`120 / 280 / 450 / 650 / 900 / 1200`。
- 收藏页现可展示：
  - 已解锁/未解锁
  - 每个收藏物获取数量（累计）
  - 文本介绍与解锁提示

### 5) 成就页面（新增）
- 已新增独立成就页与主菜单入口。
- 当前为本地计算型成就，展示“达成/进度”状态。
- 成就进度来源于本地数据（局数、最高分、收藏数量、模式解锁等）。

### 6) 输入与界面问题修复
- 主菜单 `Esc` 检测已兼容 Input System，避免旧 Input API 直接报错。
- SampleScene 选人/菜单显示逻辑已调整，主菜单专属按钮不再错误出现在非主菜单页。

## 三、当前可演示流程（建议中期答辩现场使用）
1. 进入主菜单，展示排行榜/收藏/成就/设置四个页面。
2. 进入游戏并推进到里程碑深度，拾取收藏物。
3. 返回主菜单，验证收藏数量增长与成就进度变化。
4. 展示模式选择与排行榜记录更新。

## 四、已知问题与风险
