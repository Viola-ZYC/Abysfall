# 无尽深渊（Abyssal Dive）整合策划案（可迭代版）

> 文档用途：统一项目目标、当前实现、后续扩展边界，供开发与答辩口径共用。  
> 最新更新：2026-03-12

---

## 0. 版本基线

- 项目定位：Unity 2D 纵向下坠生存跑酷 + 轻 Roguelike
- 中期结论：核心可演示闭环已完成（功能向约 80%-85%）
- 当前开发基线：**左右移动 + 下坠触敌自动反冲（无主动攻击键）**

> 注：早期 Spec 中“主动攻击触发”保留为扩展分支，不纳入当前稳定版本硬验收。

---

## 1. 核心体验目标

- 单局时长目标：3-5 分钟
- 核心博弈：
  - 风险：速度和密度上升导致容错下降
  - 收益：对齐敌人反冲可换取生存窗口与更高分
- 核心循环：
  - 主菜单 -> 开始 -> 下坠生存 -> 踩踏特殊生物获取能力 -> 结算 -> 重开/返回主菜单

### 1.1 背景与教程节奏（首次游玩）
- 背景铺垫（非CG）：莱姆误触遗迹重力机关坠入深渊，真相与能力由来藏于收集品与生物图鉴
- CG1（开局）：入口探查 -> 地面崩塌 -> 坠入黑暗，提示“深渊机关已触发，唯有下坠求生”，进入新手教程一
- 教程一：高空碎台（遗迹上层残骸），低速引导基础操作；达到指定距离进入休息平台
- CG2（休息平台）：深渊能量融入，提示“获得特殊能力”，进入新手教程二
- 教程二：晶光通道（遗迹中层），引导能力使用；抵达底部后教程结束
- 教程结束对话：返回主界面 / 开启无尽模式
- CG3（开启无尽模式）：穿过晶光通道，进入黑暗深渊，提示“新手教程已完成，无尽挑战开启”
- 正式游戏：黑暗深渊单一背景，无上限递增下坠；收集品与图鉴碎片解锁深渊背景与能力来历

## 2. 功能模块口径（当前）

### 2.1 玩家与碰撞
- 重力递增与下落上限控制
- 左右移动与边界阻挡
- 触敌反冲
- 障碍扣血 + 减速 + 障碍消失
- 高速防穿透

### 2.2 成长与能力
- 踩踏击杀“特殊生物”直接获得对应能力，并替换当前能力
- 特殊生物生成：每 100 分生成一只，从生物池随机选择
- 不同特殊生物有各自解锁位置（未遇到的不显示详情）
- 踩踏击杀后暂停游戏，弹出能力描述栏；玩家点击任意位置恢复
- 踩踏生物的移动效果与踏板砖块一致
- 能力释放改为屏幕正下方按键触发
  - 主动能力：按键可点击
  - 被动能力：按键变暗且不可点击
- 角色能力控制器已支持空跳能力（踩方块/障碍刷新）

### 2.3 菜单与HUD
- 主菜单四页面：Leaderboard / Codex（图鉴） / Achievements / Settings
- 图鉴包含四类页面：收藏品 / 障碍 / 跳板 / 生物
  - 图鉴内提供四个按钮切换，默认显示“生物”图鉴
  - 未遇到的生物显示为“?”占位，且不可点击
- HUD 已显示：分数、血量、速度（SPD）、加速度（ACC）
- HUD 增加能力按键（屏幕正下方，随主动/被动状态变化）
- 暂停与结算菜单路径完整

### 2.4 数据与解锁
- `run_progress.json` 存储分数、局数、模式、图鉴解锁与排行榜
- 模式解锁：Classic（默认）/ Hard（400分+3局）/ Abyss（900分+8局）
- 图鉴解锁以“获得/遭遇记录”为主，不再依赖固定收藏里程碑

### 2.5 动画状态驱动
- `PlayerAnimatorStateDriver` 驱动 Animator 参数：`IsJumping/IsFalling`

### 2.6 新增需求（待实现）
- 所有砖块的 Sprite、动画与显示特效补齐
- 加速度调整：初始速度慢一点、加速更明显
- 能力获取方式改为“特殊生物踩踏获取并替换当前能力”
- 新增能力按键（屏幕正下方），被动能力按键置灰且不可点
- 收集系统改为图鉴（收藏品/障碍/跳板/生物四页，默认生物）
- 设置中去掉分辨率选项，难度区分完善（不同难度差异明确）

## 5. 数值与回归口径

- 宝箱间隔默认：每 100 分
- 图鉴解锁进度以“获得/遭遇记录”为准（四类条目）
- 特殊生物生成间隔：每 100 分
- 加速度分档（随分数/深度）：<500 为 0.005；500-1999 为 0.01；>=2000 为 0.05
- 回归标准：
  - 连续 20 次“开始 -> 死亡 -> 重开”无阻断
  - 10 局中 >= 8 局在 3-5 分钟区间

## 6. 风险与保底策略

- 风险1：内容扩展引入新Bug
  - 应对：新增内容先在测试场景回归，再并入主线
- 风险2：难度曲线不稳定
  - 应对：10局模板 + 小步快调，禁止一次性大改
- 风险3：演示版不稳定
  - 应对：设封板版本，扩展项不达标时不影响主版本
